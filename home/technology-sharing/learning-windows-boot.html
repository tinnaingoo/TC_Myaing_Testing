Window boot ကိုလေ့လာခြင်း

By tinnaingoo • March 1, 2025

ကွန်ပျူတာသုံးသူအများစုမှာ Windows OS ကိုသုံးကြပါသည်။ OS မတက်တော့သည့်အခါ စိတ်ဆင်းရဲကြ ရသည်။
မိမိက ဘာတစ်ခုမှ မလုပ်တတ်လျှင် Service ကိုပို့ရလိမ့်မည်။
ရှေ့မှာတန်းစီနေသော ကွန် ပျူတာများရှိလျှင် သုံးရက်ခန့်စောင့်ပြီးမှ ပြန်ရသည်။
မိမိမြင်နေကျ interface ပြောင်းသွားသဖြင့် စိတ်ကသိကအောက်ဖြစ်ရသည်မျိုးရှိမည်။
Hardware (အထူးသဖြင့် hard disk) ကြောင့်ဖြစ်လျှင် ငွေကုန် ကြေးကျလည်း ရှိပါလိမ့်မည်။
1 TB hard disk တစ်လုံးကို ယခုလိုအချိန်၌ပင် ခြောက်သောင်းလောက်နှင့်ရနိုင် ၍ တော်ပေသေး၏ ဆိုရပါလိမ့်မည်။

ထို့ကြောင့် Windows boot လုပ်ငန်းစဉ်ကို လေ့လာကြပါမည်။
ငွေကုန်ကြေးကျခြင်းတူလျှင် hard disk က storage space ကြီးကြီးရသောကြောင့် သုံးစွဲသူအများစုနှင့် ကွန်ပျူတာဆိုင်အများစုက hard disk ကို အသားပေးနေဆဲဖြစ်သော်လည်း ကျွန်မအမြင်ဖြင့်ပြောရလျှင် OS ကို (SSD (Solid State Drive ) ပေါ်၌ ထားပြီး HD ကို data သိုလှောင်မှုအတွက် သာ သုံးသင့်သည့်အချိန်ရောက်နေပါပြီ။

မီးအားအတက်အကျပြင်းထန်သောမြန်မာပြည်တွင် ကျွန်မသုံးခဲ့ဖူးသောကွန်ပျူတာများသည် hard disk သက် တမ်းအများဆုံး ငါးနှစ်သာခံပါသည်။ HD အများစု၏သက်တမ်းမှာ သုံးနှစ်ကျော်မျှသာခံပါသည်။ ထူးဆန်းသည် မှာ Laptop ကွန်ပျူတာများမှ hard disk များဖြစ်သည်။ Hardware အတွဲအဖက်ညီ၍လား သို့တည်းမဟုတ် လျှပ်စစ်အားအတက်အကျကို ထိန်းသိမ်းမှုအားကောင်း၍လား ဆိုသည်ကို ဝေခွဲ၍မရ။ ကျွန်မ၏ တူတော်မောင် ကိုင်သော laptop ကွန်ပျူတာဆို လျှင် သက်တမ်း ၇ နှစ်ကျော်လာခဲ့ပြီ။ သို့သော် HD က ကောင်းကောင်းကြီး အလုပ်လုပ်နေဆဲဖြစ်ပါသည်။

SSD တို့ဈေးကျလာပြီး ကြံ့ခိုင်မှုပိုအားကောင်းလာသည်ဟုဆိုသော်လည်း 1 GB storage အတွက် ကုန်ကျ စရိတ်ကို (မြန်မာပြည်ဈေးဖြင့်တွက်လျှင်) SSD က HD ထက် လေးဆကျော်ဈေးပိုကြီးနေဆဲဖြစ်သည်။
နိုင်ငံတကာဈေးကွက်၌မူ SSD ၏ကုန်ကျစရိတ်က HD ထက် နှစ်ဆခွဲခန့်သာရှိပါတော့သည်။ Windows OS အတွက် SSD ကိုသုံးလျှင် installation လုပ်ငန်းစဉ်အလွန်မြန်လာသည်ကို တွေ့ရပါသည်။ HD ပေါ်တွင် Windows OS တင်စဉ်က အချိန်မိနစ်လေးဆယ်ခန့်ပေးခဲ့ရသော်လည်း SSD ပေါ်တွင် ဆယ့်ငါးမိနစ်ဝန်းကျင် ခန့်သာကြာသည်ကို တွေ့ရပါသည်။

Windows boot တက်ရန်အတွက် ယူရသည့်အချိန်ကို နှိုင်းယှဉ်လျှင်လည်း Free Storage Space ခြောက် ဆယ်ရာခိုင်နှုန်းကျန်သော SSD တွင် အချိန် ၂၅ စက္ကန့်ဝန်းကျင်သာယူပြီး အလားတူအခြေအနေအတွက် HD တွင် အချိန် ၃၅ စက္ကန့်မျှ ယူသည်ကို တွေ့ရပါသည်။ Boot တက်ရန် မြန်ခြင်းနှေးခြင်းသည် SSD နှင့် HD မတူညီမှု တစ်ခုတည်းပေါ်တွင် မူတည်ပါသလား။ ထိုသို့မဟုတ်ပါ။

BIOS နှင့် UEFI ကွဲပြားမှုတို့အပေါ်တွင် လည်း မူတည်ပါသည်။ BIOS ဆိုသည်မှာ Basic In put Output System ကို ဆိုလိုခြင်းဖြစ်ပြီး ၂ဝ၁ဝ ဝန်းကျင်အထိ ကွန်ပျူတာအများစုတွင် အသုံးပြုခဲ့ပါသည်။ ယခုအထိလည်း ကွန်ပျူတာအချို့တွင် အသုံးပြု နေဆဲ ဖြစ်ပါသည်။


UEFI မှာ Unified Extensible Firmware Interface ကို ဆိုလိုခြင်းဖြစ်ပါသည်။
ဤဘက်ခေတ် ကွန်ပျူတာ အများစုတွင် အသုံးပြုနေပါသည်။
UEFI သည်လည်း basic input/output system ကိုအထောက်အပံ့ ပေးပါသည်။
သို့သော် BIOS ကပေးစွမ်းသော အခြေခံအဆင့်ထက်ပို၍ မြင့်ပါသည်။
ထို့ကြောင့် ခေတ်ပေါ် ကွန်ပျူတာများတွင် UEFI BIOS ထည့်သွင်းပေးထားသည်ဟု ဆိုကြပါသည်။
UEFI ၏အားသာချက်များကြောင့် ကွန်ပျူ တာအများစုရှိ BIOS အား UEFI BIOS ဖြင့် တဖြည်းဖြည်း အစားထိုး အသုံးပြုလာလျက်ရှိပါသည်။

BIOS သည်ကွန်ပျူတာပရိုဂရမ်အငယ်စားတစ်ခုဖြစ်ပါသည်။ ယင်းကို motherboard ရှိ Chip တစ်ခုပေါ်တွင် ထည့်သွင်းထားပါသည်။ ကွန်ပျူတာရှိပါဝါကိုစဖွင့်လိုက်သည့်အခါ ပထမဆုံးထ၍အလုပ်လုပ်သူမှာ BIOS ဖြစ် သည်။ ကွန်ပျူတာပေါ်ရှိအခြေခံ hardware များ ကောင်းမကောင်းကို BIOS က စစ်ဆေးပါသည်။ အားလုံး ကောင်းသည်ဆိုပါမှ ရှေ့သို့ ဆက်တက်ပါသည်။
ဥပမာပြရလျှင် ကွန်ပျူတာရှိ RAM ချောင်နေပါက Windows မတက်ပါ။
BIOS အလုပ်လုပ်မှုရပ်သွားပြီး error message လာပြပါမည်။

အခြေခံ hardware များကောင်း၍ ရှေ့ဆက်တက်နိုင်ပြီဟုဆိုလျှင် BIOS က boot sector ရှိ Code များ ထ၍အလုပ်လုပ်စေရန် ဆောင်ရွက်ပါသည်။
Boot sector တွင် code များမတွေ့တော့လျှင် Windows boot မတက်နိုင်ပါ။
Boot Disk သို့မဟုတ် no operating system installed ဟူသည့် failure message မျိုး လာ၍ပြပါမည်။
ကွန်ပျူတာ၏ DVD dirve ထဲတွင် DVD အချပ်တစ်ခုထည့်ထားမိလျှင် သို့မဟုတ် Windows အတက်တွင် မင်မိုရီ စတစ်တစ်ချောင်းမေ့လျော့ပြီး စွပ်ထားမိလျှင်လည်း ယင်း error မျိုးပြတတ်ပါသည်။
DVD အချပ်မထည့်ထားမိဘဲ သို့မဟုတ် မင်မရီစတစ် ထိုးမထားမိဘဲ ထို error မျိုးလာပြလျှင် hard disk ပျက်ဖို့ များပါသည်။

HD အားဆက်သွယ်ထားသောကြိုးချောင်နေပြီး မမိတမိဖြစ်နေလျှင် သို့တည်းမဟုတ် hard disk ၌ပါဝါ ကောင်းစွာမဝင်လျှင်လည်း error message တက်တတ်ပါသည်။
ထို့ကြောင့် Windows boot မတက်ကာမျှနှင့် hard disk ပျက်ပြီဟု မယူဆသင့်ပါ။ Hard disk နောက်မှ လာထိုးသောပါဝါကြိုးနှင့် data ကြိုးကို အသေ အချာကြပ်အောင်လုပ်ပြီး ကွန်ပျူတာကို ပြန်ဖွင့်ပြီး စမ်းသင့်ပါသည်။
HD ပေါ်ရှိ boot sector သည် မတော်တဆထိခိုက်၍ပျက် စီးခြင်း သို့မဟုတ် ကွန်ပျူတာအသုံးကြာလာ၍ boot sector ချို့ယွင်းခြင်း ဖြစ်လာပါက အဆိုပါနေရာရှိ code အား BIOS သွား၍ နှိုးဆော်ခြင်းမပြု နိုင်တော့ပါ။
ထိုအခါမျိုးတွင်သာ HD ပျက်ပြီး Windows boot မတက်ခြင်း ဖြစ်ပါသည်။
HD ကိုဖြုတ်ပြီး အခြားကွန်ပျူတာအကောင်းတစ်လုံးတွင် စမ်းပါက Detect မသိတော့သည်ကို တွေ့ရပါမည်။

Boot ကိစ္စနှင့်ပတ်သက်ပြီး ပြောရလျှင် MBR နှင့် GPT လည်း ပါလာပါမည်။
MBR သည် Master Boot Record ဖြစ်သည်။ MBR သည် ကွန်ပျူတာပေါ်ရှိ partition များအကြောင်းကို ပြောသည်။
Data များထားသိုပုံကို ပြောသည်။ MBR ကောင်းနေလျှင် Device Boot ကိစ္စကိုဆက်လက်ဆောင်ရွက်သွားပါသည်။
ထိုအချိန်တွင် BIOS က သယ်ဆောင်လာသော screen အမဲပျောက်သွားပြီး ရောင်စုံ Windows screen တက်လာသည်ကို တွေ့ရပါမည်။ MBR သည် အားသာချက်ရှိသလို အားနည်းချက်လည်းရှိသည်။ MBR သည် partition တစ်ခု အတွက် 32 bit data ကိုသာထိန်းသိမ်းနိုင်သည်။ ထို့ကြောင့် MBR ကိုအခြေခံ၍အလုပ်လုပ်သော ကွန်ပျူတာ တစ်လုံးတွင် partition တစ်ခုကို 2 TB ထက်ပိုကြီးအောင် ထား၍မရပါ။

ယင်းကိစ္စသည် ၁၉၈ဝ ပြည့်လွန်နှစ်များဆီကဆိုလျှင် ပြဿနာ မဟုတ်ပါ။
Hard disk တစ်လုံး၏ အရွယ်အစား သည် 1 GB ပင်မရှိသော ကြောင့်ဖြစ်သည်။
ယခုအချိန်၌မူ ငွေကြေးတတ်နိုင်လျှင် 10 TB ခန့်ရှိသော HD သုံးလာနိုင်ပြီဖြစ်ရာ MBR က ပြဿနာတစ်ခုဖြစ်လာပါသည်။
ထို့ပြင် MBR ၏အားနည်းချက်မှာ ကွန်ပျူတာ တွင် partition လေးခုထက် ပို၍ ပိုင်းခွင့်မရှိခြင်းဖြစ်သည်။
ယင်း partition များ၌ပင်လျှင် နှစ်ခုကို extended partition, Logical partition အဖြစ်ထားရပါသည်။
ထို့ကြောင့်ခေတ်ပေါ်တွင် ကွန်ပျူတာကြီးများတွင် GPT (GUID partition table) ကို အသုံးပြုလာပါသည်။
GPT ၏အားသာချက်မှာ hard disk တစ်လုံးပေါ်တွင် partition ၁၂၈ ပိုင်းအထိ ပိုင်းခွင့်ရှိခြင်းဖြစ်သည်။
ထို့ပြင် GPT သည် Zettabyte အရွယ်အစား hard disk အထိလည်း အထောက်အပံ့ပေးပါသည်။
